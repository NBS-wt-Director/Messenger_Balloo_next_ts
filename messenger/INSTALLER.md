### 3. **Yandex OAuth & Disk** ☁️

#### Какие доступы нужны:

```
oauth:login          — доступ к логину пользователя
oauth:avatar         — доступ к аватару
oauth:email          — доступ к email
cloud_api:disk.read  — чтение Яндекс.Диска
cloud_api:disk.write — запись на Яндекс.Диск
```

#### Как получить:

1. Перейдите на https://oauth.yandex.ru/
2. Создайте новое приложение
3. Укажите Redirect URI:
   ```
   http://localhost:3000/api/auth/yandex/callback (dev)
   https://ваш-домен.ru/api/auth/yandex/callback (prod)
   ```
4. Выберите права доступа (см. выше)
5. Скопируйте **Client ID** и **Client Secret**
6. Для токена Disk перейдите по ссылке:
   ```
   https://oauth.yandex.ru/authorize?response_type=token&client_id=ВАШ_CLIENT_ID
   ```
7. Скопируйте токен из URL браузера

#### Проверка аккаунта:

- ✅ **Кнопка "Проверить"** — проверяет токен Yandex Disk
- ✅ **Валидный аккаунт** — должен содержать `o8eryuhtin` в логине
- ⚠️ **Предупреждение** — если логин не содержит `o8eryuhtin`

## ✅ Процесс установки

1. **Заполните все обязательные поля** (помечены `*`)
2. **Проверьте Yandex OAuth** (кнопка "Проверить")
3. **Выберите опции** (очистка, тестовые аккаунты, рандом)
4. **Нажмите "Установить"**
5. **Дождитесь завершения** (прогресс-бар)
6. **Перезапустите сервер**
   ```bash
   # Остановите текущий процесс
   Ctrl+C
   
   # Запустите снова
   npm run dev
   # или для продакшена
   npm start
   ```

## 📁 Создаваемые файлы

### 1. `.env.local`
```bash
# Server
NEXT_PUBLIC_SERVER_URL=https://ваш-домен.ru
NODE_ENV=production
PORT=3000

# Security
NEXTAUTH_SECRET=сгенерированный_секрет

# Yandex OAuth
YANDEX_CLIENT_ID=ваш_client_id
YANDEX_CLIENT_SECRET=ваш_client_secret
YANDEX_DISK_API_URL=https://cloud-api.yandex.net/v1/disk
YANDEX_DISK_TOKEN=ваш_токен

# Push Notifications
VAPID_PUBLIC_KEY=сгенерированный_ключ
VAPID_PRIVATE_KEY=сгенерированный_ключ
VAPID_SUBJECT=mailto:admin@ваш-домен.ru

# Email (optional)

SMTP_PORT=587
SMTP_USER=почта@yandex.ru
SMTP_PASS=пароль

# Admin
ADMIN_EMAIL=admin@ваш-домен.ru
```

## 🆘 Устранение проблем

### Yandex OAuth не проверяется
- Проверьте что Client ID и Secret без пробелов
- Убедитесь что redirect URI совпадает с указанным в приложении
- Проверьте что токен Disk скопирован полностью

### VAPID ключи не работают
- Перегенерируйте ключи: `npx web-push generate-vapid-keys`
- Обновите `config.json`
- Перезапустите сервер

### Ошибка при создании файлов
- Проверьте права доступа к папке проекта
- Убедитесь что нет конфликта с существующими файлами
